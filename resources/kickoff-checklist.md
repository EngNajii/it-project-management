# ✅ Project Kickoff Checklist

Use this checklist before and during your project kickoff to ensure nothing is missed.

---

## 📌 Before the Kickoff Meeting

### Governance & Documentation
- [ ] Project Charter drafted and approved by sponsor
- [ ] Project code / ID assigned
- [ ] Project added to the portfolio tracker
- [ ] Stakeholder list compiled
- [ ] Communication plan drafted

### Team
- [ ] Project team identified and confirmed
- [ ] Roles and responsibilities assigned (RACI complete)
- [ ] Team access and tools provisioned (Jira, Confluence, Slack, etc.)
- [ ] Onboarding materials shared with new team members

### Planning
- [ ] High-level scope defined
- [ ] Initial schedule / milestone plan created
- [ ] Budget estimate approved
- [ ] Risk register started (at least top 5 risks identified)
- [ ] Dependencies identified

---

## 🚀 During the Kickoff Meeting

- [ ] Introductions (team members, sponsors, stakeholders)
- [ ] Present project vision and objectives
- [ ] Review scope: what's in and out
- [ ] Walk through high-level timeline
- [ ] Clarify roles (who owns what)
- [ ] Review communication cadence (meetings, reports, escalation paths)
- [ ] Review tools and processes (where to log issues, how to request changes)
- [ ] Open Q&A – address concerns and ambiguities
- [ ] Celebrate the start! 🎉

---

## 📋 After the Kickoff Meeting

- [ ] Meeting notes distributed within 24 hours
- [ ] Action items logged and assigned
- [ ] Project plan baselined in PM tool
- [ ] First status report scheduled
- [ ] Project repository / shared drive set up
- [ ] Kickoff deck / materials archived

---

## 💡 Tips

> **Keep the kickoff focused** – it should run 60–90 minutes max. Deep dives happen in separate working sessions.

> **Set the tone** – A good kickoff builds team confidence and alignment. Be enthusiastic and clear.

> **Document everything** – What gets agreed in the kickoff becomes the baseline for the entire project.
